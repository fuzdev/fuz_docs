auth/account_queries.ts

Account and actor database queries.

Provides CRUD operations for the account and actor tables. For v1, every account has exactly one actor (1:1).

query_admin_account_list
#

auth/account_queries.ts view source

(deps: QueryDeps): Promise<{ account: { id: string & $brand<"Uuid">; username: string; email: string | null; email_verified: boolean; created_at: string; updated_at: string; updated_by: (string & $brand<...>) | null; }; actor: { ...; } | null; permits: { ...; }[]; pending_offers: { ...; }[]; }[]>

List all accounts with their actors, active permits, and pending inbound permit offers for admin display.

Uses 4 flat queries instead of N+1 per-account loops. Pending offers surface the "offer pending — awaiting acceptance" UX without a second round-trip; message is intentionally excluded (cross-admin visibility of grantor notes would expand beyond what the audit log discloses).
