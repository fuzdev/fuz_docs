palette_glosses

variable_data.ts view source

Record<"g" | "b" | "a" | "c" | "d" | "e" | "f" | "h" | "i" | "j", { color: string; binding?: "accent" | "positive" | "negative" | "caution" | "info" | "neutral" | undefined; }> import {palette_glosses} from '@fuzdev/fuz_css/variable_data.js';

Letter glosses: each palette slot's default color plus its default intent binding where one exists. Display data for docs and the theme editor — themes recolor the slots, so these describe the defaults, not promises. The neutral binds like an intent (--hue_neutral: var(--hue_f)) despite being absent from intent_variants.
