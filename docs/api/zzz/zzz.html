zzz

13 modules

  • zzz/build_info.ts

  • zzz/cli_config.ts

    zzz CLI configuration.

    Manages CLI-specific configuration stored at ~/.zzz/config.json.

    The CLI config uses the zzz_config_ prefix for all fields to make the source self-documenting in code.

  • zzz/cli.ts

    zzz CLI argument parsing and help.

    Thin wrapper around argv_parse + extract_global_flags.

  • zzz/cli/cli_args.ts

    CLI argument parsing utilities for zzz.

    Provides zzz-specific dispatch and subcommand routing. Generic parsing utilities come from @fuzdev/fuz_app/cli/args.js.

  • zzz/cli/cli_help.ts

    CLI help generation and command metadata.

  • zzz/cli/schemas.ts

    Per-command Zod schemas for CLI argument validation.

    Centralized here to avoid importing runtime deps (e.g., server.ts) when only schemas are needed (e.g., for help generation).

  • zzz/commands/daemon.ts

    zzz daemon commands (start, stop, status).

    The zzz CLI runs in Deno, so daemon start uses the Deno server entry point.

    Routing (zzz daemon start|stop|status) is handled by create_subcommand_router in main.ts.

  • zzz/commands/init.ts

    zzz init command.

    Initialize zzz configuration (~/.zzz/).

  • zzz/commands/open.ts

    zzz open command (default command).

    Opens the zzz browser UI, auto-starting the daemon if needed. Handles: zzz, zzz <file>, zzz <dir>.

  • zzz/commands/status.ts

    zzz status command.

    Show current system state (daemon status, loaded workspaces, watched repos).

  • zzz/log.ts

    zzz CLI logger — Logger + CLI semantic methods.

  • zzz/main.ts

    zzz CLI entry point.

  • zzz/runtime/types.ts

    Runtime type alias for zzz.

    The full runtime interface is shared via RuntimeDeps from fuz_app. ZzzRuntime is a project-local alias used throughout the CLI code.

    Functions should accept narrow *Deps interfaces from @fuzdev/fuz_app/runtime/deps.js, not the full ZzzRuntime.
