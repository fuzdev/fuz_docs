Canonical action spec list — fuz_app's protocol actions (heartbeat, cancel) plus zzz-owned specs. Single source of truth for codegen and runtime registries (Frontend.action_registry, Backend.action_registry).

Codegen helpers default-filter protocol actions out of typed surfaces (FrontendActionsApi, BackendActionHandlers, ActionMethod, etc.) since they're dispatcher-owned. Server WS registration pairs protocol specs with fuz_app-shipped handlers via protocol_actions; iteration over all_action_specs skips them via is_protocol_action_method to avoid double-registration. Adding a future protocol action requires no consumer edit — fuz_app's protocol_action_specs is the upstream canonical bundle.
